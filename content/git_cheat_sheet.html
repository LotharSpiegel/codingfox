Merge vs. Rebase

  • Merge: Fügt Änderungen von einem Branch in einen anderen ein, indem ein neuer Merge-Commit erstellt wird.
  • Rebase: Integriert Änderungen, indem die Commits neu angewendet werden, sodass die Historie linear bleibt.

Ignore Files

  • After adding files to `.gitignore` after they were already committed: `git rm -r --cached some_file.txt`
